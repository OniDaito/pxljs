PXL.js is a WebGL Engine. It is simple to use and has various functions for the most common processes you need to code when writing WebGL.

What does it offer?

PXL.js has the following features:

  • Easy to use Node / Scenegraph / Geometry design
  • UberShader and Materials
  • OBJ and MD5 model support with bone skinning
  • Basic lighting with Phong materials and Shadowmapping
  • Texture and Framebuffer Object Management
  • Large math library including curves and interpolations
  • Promises, requests and asynchronous support built-in
  • CoffeeScript and Javascript versions
  • Offline compiler for your own ubershaders
  • Easy ways of ignoring scene-graph and ubershader if you need specialist shaders

How can I use it?

You can clone from github, download using npm or by downloading the latest zip. Take a look at the documentation to get a feel for how PXLjs works. From there, you can take a look at the examples and their code to see how it fits together. There is also the API documentation that describes all the various functions.

Where is it used?

An early version of PXL.js is used to render the Equatorie at Cambridge Unviersity.

What is the current status of PXL.js?

The current version of PXL.js is 0.1.0. There are many more features I'd like to add including:

  • Support for WebGL2.0
  • MD5 Animation support
  • Binary formats of model data
  • Caching of the scenegraph for speed
  • VSM Shadowmaps and Shadow Volumes
  • Easy to use post / screen space effects such as SSAO
  • More procedural shapes
  • More materials
